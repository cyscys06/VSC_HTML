도메인 {
    도메인 = 시스템이 구현해야 하는 핵심 비즈니스 업무 영역을 말함
    ex) 로또 발매기: 최상위 도메인
    -> 로또 발매, 번호 비교, 총 상금액 계산 등: 하위 도메인
    
    회원 이용 사이트: 최상위 도메인
    홈 화면(로그인 전)
    홈 화면(로그인 후)
    회원가입
    상품관리
    -> 하위 도메인
}

쿠키 {
    웹 브라우저: 인터넷의 웹사이트 정볼를 찾아 표시해주는 응용 소프트웨어
    -> 사용자가 url을 입력하면 브라우저는 웹 서버에서 해당 url에 해당하는 페이지를 요청함
    -> 이후 그 페이자의 컨텐츠를 사용자에게 제공하는 역할
    
    쿠키: 웹사이트에 접속할때 웹사이트 서버가 만드는 데이터 조각
    -> 웹 서버에서 만들어진 쿠키는 웹 브라우저가 저장하고 있음(데이터 조각을 저장)
    -> 사용자가 동일한 서버에 http를 재요청할때, 웹 브라우저가 http 요청에 쿠키를 같이 보냄
    -> 서버는 쿠키를 통해 처음 보낸 http, 재요청한 http 요청을 비교
    -> 두 요청이 같으면 사용자의 로그인 상태를 유지, 다르면 로그인 해제하는 방식
    -> 사용자의 로그인 상태를 유지할 수 있는 기능(같은 사용자가 보내는 요청이면 쿠키가 같을것이므로)
    
    영속 쿠키, 세션 쿠키 {
        영속 쿠키: 만료 날짜를 입력하면 해당 날짜까지 유지
        세션 쿠키: 만료 날짜를 생략하면 브라우저 종료시 까지만 유지
    }

    @CookieValue 어노테이션 사용해서 편하게 쿠키 조회 가능

    Cookie idCookie = new Cookie("memberId", String.valueOf(loginMember.getId()));
    // 쿠키 생성
    response.addCookie(idCookie);
    // HttpServletResponse 객체(클라이언트에 다시 보내는 응답)에 쿠키를 추가(회원 id를 쿠키에 담음)

    로그아웃할때의 쿠키 조정 {
        로그아웃할때 쿠키의 종료날짜를 0으로 조정
    }

    쿠키 단점 {
        웹사이트 개발자도구 들어가면 쿠키 정보가 공개돼있음
        -> 공개된 쿠키의 정보에 중요한 정보가 들어있으면 해커에게 털릴수 있음
        
        해결: 쿠키에 중요한 값을 노출시키지 않고, 사용자별 토큰을 생성해서 해커가 토큰을 예측하지 못하게 하기
        -> 토큰 만료시간 짧게 설정하거나 해킹이 의심되면 토큰을 강제로 종료시키는것도 가능
    }
}
